1      Scope

The scope of this document is to provide step-by-step procedure or instructions on the usage of each functionality of the Land Management System web application.

1.1    Overview

Chennai Port, the third oldest port among the 12 major ports of India, is an emerging hub in the East Coast. This gateway port for all cargo has completed 128 years of glorious service to the nations maritime. The Chennai Port has Terminal Shunting Yard and runs its own railway operations inside the harbor on the East Coast. The port has railway lines running up to 68kms and handles 25% of the total volume of the cargo, 4360 rakes (239412 wagons) during 2009-10 trade. In the first couple of years the port registered traffic of 3 lakh tons of cargo handling 600 ships. The port with three Docks, 24 berths and draft ranging from 12m to 16.5m has become the hub port for Containers, Cars and Project Cargo in the East Coast. The port has handled an all-time high of 61.06 Million tons of cargo registering an increase of 6.2% over previous year. An increase of 10.14% in handling of cars from 273917 Units in the year 2009-10 when compared with 248697 Units in the year 2008-09 and an increase of 6.39% in handling of containers from 1143373 TEUs in the year 2008-09 to 1216438 TEUs in the year 2009-10. The long term plan for Chennai Port envisages handling of 4Cs i.e., Containers, Cars, Cruise and Clean Cargo.

3      Features

 

Land Management System, a Geographical Information System (GIS) based web and mobile application for Chennai Port Trust provides the necessary tools/functionality to:

         Manage the users and their roles to access the application

         Manage the existing and new spatial data

         Visualize the digital data geographically on the map

         Show the up-to-date status information of the plots and buildings

         Provide the capability to book the plots for lease over internet

         Generate operational and financial reports as and when required.

Based on the requirements, Land Management System has been designed and developed with various features. The application has been divided into the following modules:

            User Management

            Role Management

            Data Management

            Map

            Reports

User Management:

The User Management module primarily manages the internal port users authentication and their roles to access the application. By default the Administrator role has complete access to the application. Administrators responsibilities include registering new users, modifying the user details and deactivating the users.

Role Management:

Role Management module manages the user roles and access rights to the application. Based on the requirement the following roles are included in the application:

Role

Access

Administrator

All the modules

Port Officer/Port Staff

Map module for viewing only

Leasee

Map module and related functionality

Public User/Visitor

Map module for viewing only

If required, the Administrator can create new roles or deactivate the existing roles as and when required.

Data Management:

Data Management module manages geo-spatial data (shape files) which can be imported into the application as an additional asset/layer. This feature enables the Administrator to include new assets/layers of the port into the application to view them on the map.

Geo referenced shape files (zipped) are the inputs for this capability. While importing the zipped shape files, it is mandatory to provide required details such as workspace, store, layer, and coordinate system to publish the assets/layer. The assets/layers imported into the application can be viewed on the map.

 

Map:

Map module deals with the visualization of all geo-spatial data of the Chennai Port

            Various top class base maps such as Open Street Map, Bing Map, Bing Road Map, Bing Label Map, Street Map, Carto DB Map, OSM Satellite Map and Satellite Map are provided in the application to visualize the port area in different aspects. Base map can be toggled as and when required.

            Nearly 100+ assets/layers are accurately overlaid on top of the base map. These layers can be viewed in multiple scales with on & off display.

            Features in each layer such as plots/buildings/docks etc., are symbolized as per the standards provided by the Chennai Port authority.

            For easy navigation around the port area on the map, navigation tools such as zoom in, zoom out, pan, scale, coordinates at mouse position etc. are provided.

            Feature information can be viewed by clicking the mouse on any plots or buildings on the map. It shows the available information like name, area, etc., of the selected plot or building including the attached image and document.

            With the capability of Upload Image and Upload Document, it is very easy to attach any .jpg, .png, .doc, .pdf file to the selected feature. Also, the preview of the attached image is displayed in the provided widget.

            A Table view is provided to display the booked plots history of a particular leasee. This information can be seen only when the concerned party is logged into the application.

            The Legend shows the color coding for the booked, occupied, available plots and buildings.

            Capabilities of Attribute and Spatial queries are provided to search and fetch the desired features from the spatial data and display on the map. This query can be performed by using either the known attributes/information of the desired feature or by using spatial query - within the specified distance from a well-known location.

            Feature selection capability is provided to select single or multiple features on the map to perform further activities.

            Registered party or a guest can send a request to the Chennai Port authority to book an area in open storage plots or in a building in the port. Registered party or a guest can fill up and submit their details in the provided Application Form for an Allotment.

Reports:

Reports module demonstrates the operational and financial information of every plot and building. The operational and financial information is being maintained by the ERP system of Chennai Port Authority.

Registered parties can see their own plots operational and financial information at any point of time by using the web and mobile applications.

Administrator can see the total summary of the plots by using the summary tool that shows the pie chart of the overall percentage of the booked, occupied and available plots.

5      Troubleshooting

This section illustrates possible errors or problems that may occur while using the application

            The following error may occur due to following reasons:

1.     If the Tomcat server is down

2.     In case of network issue

3.     If user tries to access application with improper URL.

Solution:

User needs to check their network availability and enter valid application name in URL: ChennaiPort (case-sensitive) or contact administrator to restart the Tomcat server.

 

            HTTPS Status 404 occurs on any unexpected exception/error in the application.

Solution:

Contact administrator to restart the server.

            The home page of the application loads as shown in the following figure in case of internet unavailability.

Solution:

Ensure that your system is connected to the internet.

            The home page of the application loads as shown in the following figure in case of database unavailability.

Solution:

Contact the administrator to ensure database availability.

            Map page: GPS Location does not zoom to the current location if the user browser is under insecure origin.

Solution:

To use this feature, user should consider switching the application to a secure origin, such as HTTPS.
